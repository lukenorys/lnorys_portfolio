Horiseon

Refactor Code

Horiseon needed their website to be more accesible.
The goal was not only to add "alt" descriptions to images, but also scout the code.
The style.css file had the same code written multiple times. Through the process of refactoring,
the DRY (don't repeat yourself) method was used and the code was cleaned up!
The entire process gave the website a Lighthouse "Best Practice" score of 100 and 'Accessibility' score of 93.
In the end their search engine optimization took a great turn for the better.
I also added in a hex-color theme matching Favicon as an extra feature. Click the image to take a look!
